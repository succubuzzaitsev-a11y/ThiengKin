Best-of-Breed — ยืมจุดเด่นจากแต่ละแอป

แทนที่จะสู้กับ Wongnai/Google ตรงๆ — เลือกเฉพาะ "จุดเด่น" ของแต่ละเจ้ามารวมไว้ในที่เดียว

🥇 Wongnai (เบอร์ 1 ไทย)

Phase 1.5

จุดเด่นที่ยืม:

  • Categories ละเอียดแบบ local (ก๋วยเตี๋ยว, ส้มตำ, ข้าวซอย, น้ำตก, ปิ้งย่าง)
  • "555 อันดับ" → เพิ่ม "User Votes" กด "อร่อยจริง" เพิ่ม AI weight
  • "แนะนำสำหรับคุณ" → AI เรียนรู้จาก history

ที่ไม่เอา:

  • โฆษณา, booking, delivery (คนละ business model)
  • Member rewards (ต้องทีมใหญ่)

🤖 Google Maps + Gemini AI

Phase 2

จุดเด่นที่ยืม:

  • AI Review Summary — "เด่น: ก๋วยเตี๋ยวเนื้อ / ขาด: ที่จอดรถ"
  • Contextual Attributes — filter: WiFi, ที่จอด, Pet-friendly, เปิดดึก
  • "Ask Maps" chat — ถาม "อยากกินอะไรดี" ได้ (Gemini Flash)
  • Personalization จาก save/favorite

ที่ไม่เอา:

  • "Inspiration curated" (Google เฉพาะเจาะจงเกินไป)
  • Track ทุก search (privacy issue)

🍜 Kinraidee (กินไรดี) + Jaew Pa Gin

Phase 1

จุดเด่นที่ยืม:

  • "วันนี้กินอะไรดี?" random button (Kinraidee)
  • Quick presets — 1 tap ได้ร้าน (Jaew Pa Gin style)
  • Calorie counter (optional, ถ้ามีข้อมูลเมนู)

ที่ไม่เอา:

  • Random menu เฉยๆ (ไม่มี GPS/distance)

📅 Hungry Hub + OpenRice

Phase 3

จุดเด่นที่ยืม:

  • Multi-language — Thai-first + English toggle (Hungry Hub 15 langs)
  • Price range filter (OpenRice) — $ / $$ / $$$ / $$$$
  • Deep link ไป Hungry Hub สำหรับ booking (ไม่ทำเอง)

ที่ไม่เอา:

  • Booking/Deals เอง (ทำยาก, scale ไม่ได้)

🌍 Dine AI + Place Review Analyzer + Umamii

Phase 3

จุดเด่นที่ยืม:

  • Aspect scores — รสชาติ/บริการ/ความคุ้ม/บรรยากาศ (Review Analyzer)
  • "เมนูแนะนำ" — Dine AI วิเคราะห์รีวิวแนะนำเมนู
  • Social discovery (Umamii) — "เพื่อนคุณชอบร้านนี้"

ที่ไม่เอา:

  • Social graph แบบเต็ม (ต้อง community ใหญ่)

🏘️ เฉพาะ "เที่ยงกิน" — ต่างจังหวัด

จุดต่างที่ไม่มีใครเหมือน:

  • "คนท้องถิ่นแนะนำ" — badge พิเศษสำหรับร้านที่ user ในจังหวัด vote
  • "เปิดเช้าตี 4-5" — ตลาดเช้า ก๋วยเตี๋ยวเช้า (เฉพาะ ตจว.)
  • "ร้านริมทาง/ริมปั๊ม" — สำหรับคนขับรถทางไกล
  • "ของฝากประจำจังหวัด" — น้ำพริก แหนม หมูยอ ของเฉพาะถิ่น
  • "เส้นทางทัวร์อาหาร" — เชียงใหม่ 1 วัน / ภูเก็ต 1 วัน
  • Travel mode — "กำลังไปเชียงใหม่" preload ทั้งจังหวัด offline

Setup & Refresh Workflow

🛠️ ครั้งแรก (one-time setup)

  1. 1. สร้าง Foursquare API key

    ฟรี · ไม่ต้องใส่บัตรเครดิต

  2. 2. รัน setup script

    node setup.mjs --lat 13.7563 --lng 100.5018 --radius 10000

  3. 3. Filter + เก็บลง Room DB

    rating ≥ 4.0, has reviews, ในรัศมี 10 กม.

  4. 4. Verify ในแอป

    เปิดแอป · เห็นร้านครบ · กดเปิด Maps ได้

🔄 Refresh (ทุกสัปดาห์/เดือน)

  1. 1. รัน refresh script

    node setup.mjs --refresh

  2. 2. Diff กับข้อมูลเดิม

    เพิ่มร้านใหม่ · อัพเดท rating · ลบร้านปิด

  3. 3. Push to Room DB

    Auto-merge ไม่ทับ favorites

  4. 4. (Phase 2) Sync ผ่าน Supabase

    Realtime ไปอีกเครื่องอัตโนมัติ

6-Week Build Plan (Travel Mode First)

  1. Week 1 — Data Layer (เชียงใหม่ เป็น pilot)

    Setup script (Foursquare) · Room DB schema รวม corridor_cache + routes · เก็บ 100-300 ร้านเชียงใหม์ + ร้านดังริมทาง (ทางหลวง + ทางด่วน) · Manual seed: ของฝาก + ร้านเช้า + ปั๊มฯ

  2. Week 2 — Travel Mode Core (หัวใจของแอป)

    GPS permission + auto-detect driving · Province picker (เชียงใหม์) · Travel Mode UI (big buttons) · "จะไปไหน?" input · Corridor query · Top 5-10 จุดแวะ + ETA + detour · Deep link Google Maps

  3. Week 3 — AI + Wongnai-style categories

    AI ranking formula 40/30/15/10/5 (ปรับ weight ตาม mode) · Categories แบบ local (ก๋วยเตี๋ยว/ส้มตำ/ข้าวซอย/น้ำตก/ปิ้งย่าง) · "วันนี้กินอะไรดี?" random · Quick presets 5-6 อัน

  4. Week 4 — Rural Features + Offline Corridor

    "คนท้องถิ่นแนะนำ" badge · Contextual Attributes (WiFi/ที่จอด/Pet/เปิดดึก/รับบัตร) · "เปิดเช้า" "ร้านริมทาง" "ของฝาก" presets · Pre-load corridor ทั้งเส้นทาง offline

  5. Week 5 — Polish + Driving Test + Ship

    Favorites + Share · User votes ("อร่อยจริง") · AI Personalization · Test จริงขับรถกรุงเทพ→เชียงใหม่ · APK smoke test · Ship

  6. Week 6 — Add Province 2 + Phase 2 เริ่ม

    เพิ่มจังหวัดที่ 2 · เริ่ม Phase 2: Google Places via Edge Function (ถ้าต้องการร้านเล็กร้านซอยเพิ่ม)
